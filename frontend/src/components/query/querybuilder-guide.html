SqBd Search Builder - User Manual

The SqBd Search Builder is a tool designed to help you construct complex search queries for various Query Forms. It provides multiple ways to input your search terms and combine them using boolean operators, allowing you to tailor your searches more effectively.

Key Features

  • Multiple Input Methods:
    • Short Form Query Builder
    • Keyword Input
    • File Upload
  • Boolean Operators: Combine terms using AND, OR, and NOT operators.
  • Query History: Track and reuse previous queries.
  • Downloadable Query Form: Download designed queries for each form.
  • Query Form Selection: Choose target Query Forms for query building.
  • "Add To History" Button: Add current query to history list.
  • Flexible Editing: The query text area allows direct editing of the query.
  • Copy to Clipboard: Easily copy queries to your clipboard.

User Interface Overview

The interface is divided into the following key sections:

  • Input Query: This section is where you enter your search terms. It includes three different input methods, and a query text box.
  • Query History: A table that records all of the queries and its sequence and has buttons for clearing history and download history.
  • Select Query Forms: This is where you select which Query Forms you would like your query to be Collated to.
  • Collated Queries: This section displays your Collated queries for the selected Query Forms.

Detailed Instructions

1. Input Query Section

This section is further divided into the following three input method sections:

Short Form Query Builder

  • Adding Input Terms:
    • Select tag from the "field-select" dropdown menu which include "Title/Abstract", "Text Word", and "MeSH Terms".
    • Enter your search term into the input box.
  • Adding Multiple Terms:
    • To add multiple terms:
      • Click on the "Add Field" button after adding a new term.
      • A new input field will be added.
      • You can add an identifier tags and a term to the input field, like in the previous step.
  • Selecting Boolean Operators:
    • Choose between "AND", "OR", and "NOT" using the radio button options. This selection will be used to combine the input terms.
  • Clearing Input Fields:
    • Click "Clear" button to clear all input fields and the query box.

Keyword Input

  • Enter keywords separated by spaces into the text input box. The system will automatically combine these keywords with the "AND" operator.
  • Click "Submit Keywords" to add the new search terms to the query box.

File Upload

  • Upload a .txt file. The content of this file will be loaded into the query input box.
  • The file content will directly uploaded and shown in the query box and also in history.

Query Input Box

  • The query text area shows your current query based on your inputs. You can also directly edit the content of the query here.
  • You can use "Copy", "Paste", "Clear", "Undo", and "Redo" buttons under the query box.
    • Copy: copy the current query string from the text area to clipboard.
    • Paste: paste a copied query string to query text area.
    • Clear: clears all input in the query text area.
    • Undo: moves to the previous recorded query text area string.
    • Redo: Moves forward to the next record in the query text area history.
  • Adding Current Query to History:
    • Use "Add To History" button to add current query from text area to query history.

2. Query History Section

  • Viewing History: All queries added are listed with a sequence number.
  • Clearing History: Click "Clear History" to remove all the saved query from history and local storage.
  • Downloading History: Click "Download History" to export your query history as a .txt file.
  • Adding History to Query Box:
    • Click on the query entries on the history table to add that query string to the query box. The query string will be added to query box while also enclosed in parenthesis using the selected boolean operator.

3. Select Query Forms Section

Check the boxes next to the Query Forms you want to Collate your query for (PubMed customized short and extended forms, Edited keywords, and PICO parameters). You have to select at least one Query Form in order to Collate query, otherwise, an error message will be shown.

4. Collate and Download Functionalities

  • Collating/Building:
    • Click "Collate/Build" button to view the Collated results.
  • Download for each Collated Query Form:
    • Each Collated query will have a "Copy" and "Download" button.
    • Copy: Click "Copy" to copy Collated query to the clipboard.
    • Download: Click "Download" to download the Collated query as a .txt file.
  • Download All Collated Queries:
    • Click "Download All" to download all Collated queries as a single .txt file.

Building Complex Queries

Here's how to build complex, nested queries using the tools and functionalities offered by SqBd. For example, let's assume we have to create the following search query:

((challenge[tiab] OR challenge[tiab] OR difficulty[tw]) AND ("health services accessibility"[mesh] OR access[tiab])) AND ("paramedic service"[mesh] OR "ambulance service"[mesh] OR ambulance[tiab] OR ambulance[tiab] OR "emergency transport"[tw]) NOT ("air ambulance"[tiab] AND "HEMS"[mesh])
  1. Building the First Concept:
    • Use the Short Form to build challenge[tiab] OR challenge[tiab] OR difficulty[tw]:
      • In the first input field, select "Title/Abstract", add challenge to term box. Click "Add Field" button.
      • Click "OR" radio button and select "Title/Abstract", and add challenge to the term box. Click "Add Field" button.
      • Click "OR" radio button, select "Text Word" tag, and add difficulty to term box, and then click "Add Field" button.
    • Click on "Add To History" button to add this to history.
  2. Building the Second Concept:
    • Use the Short Form to build ("health services accessibility"[mesh] OR access[tiab])
      • In the first input field, select "MeSH Terms", add "health services accessibility" to term box. Click "Add Field" button.
      • Click "OR" radio button, select "Title/Abstract" tag, add access to term box, and click "Add Field" button.
    • Click on "Add To History" button to add this to history.
  3. Building the Third Concept:
    • Use the Short Form to build ("paramedic service"[mesh] OR "ambulance service"[mesh] OR ambulance[tiab] OR ambulance[tiab] OR "emergency transport"[tw])
      • In the first input field, select "MeSH Terms" tag, and add "paramedic service" to term box, click "Add Field" button.
      • Click "OR" radio button and select "MeSH Terms" tag, and add "ambulance service" to term box, click "Add Field" button.
      • Click "OR" radio button and select "Title/Abstract" tag, and add ambulance to term box, click "Add Field" button.
      • Click "OR" radio button, and add ambulance to term box, click "Add Field" button.
      • Click "OR" radio button and select "Text Word" tag, and add "emergency transport" to term box, click "Add Field" button.
    • Click on "Add To History" button to add this to history.
  4. Building the Fourth Concept:
    • Use the Short Form to build ("air ambulance"[tiab] AND "HEMS"[mesh])
      • In the first input field, select "Title/Abstract" tag, and add "air ambulance" to term box, click "Add Field" button.
      • Click "AND" radio button and select "MeSH Terms" tag, and add "HEMS" to term box, click "Add Field" button.
    • Click on "Add To History" button to add this to history.
  5. Combining Concepts:
    • Click on first history entries on history table, so that it appear on query box, add. You can verify the query in query box (challenge[tiab] OR challenge[tiab] OR difficulty[tw])
    • Click on "AND" radio button.
    • Click on the second history entry so it adds to query box, and verify the query box which should be (challenge[tiab] OR challenge[tiab] OR difficulty[tw]) AND ("health services accessibility"[mesh] OR access[tiab])
    • Now, Click on "Add To History" button.
    • Click on the third history entry so it adds to query box, and verify the query box which should be ("paramedic service"[mesh] OR "ambulance service"[mesh] OR ambulance[tiab] OR ambulance[tiab] OR "emergency transport"[tw])
    • Click "NOT" radio button
    • Click on the fourth history entry so it adds to query box, and verify the query box which should be ("paramedic service"[mesh] OR "ambulance service"[mesh] OR ambulance[tiab] OR ambulance[tiab] OR "emergency transport"[tw]) NOT ("air ambulance"[tiab] AND "HEMS"[mesh])
    • Now Click on "Add To History" button.
  6. Combining All Concepts:
    • Click on the first history entry to so that it shows up in the query text area ((challenge[tiab] OR challenge[tiab] OR difficulty[tw]) AND ("health services accessibility"[mesh] OR access[tiab])).
    • Click on "AND" radio button.
    • Click on the second to the last query which should be ("paramedic service"[mesh] OR "ambulance service"[mesh] OR ambulance[tiab] OR ambulance[tiab] OR "emergency transport"[tw]) NOT ("air ambulance"[tiab] AND "HEMS"[mesh])

You will have the combined and nested query in your query text area. Then you can submit to Collate.

Tips for Effective Use

  • Start Simple: Begin by building small parts of your query and gradually combine them to form complex nested structure.
  • Use History Wisely: Utilize the history functionality to avoid retyping previously constructed queries.
  • Clear Formatting: Ensure your query is clear and well-formed with parenthesis, tags, and the boolean operators, this is particularly important when using the copy/paste or manual edit option.
  • Combine Methods: Mix different input methods for a balanced approach.
